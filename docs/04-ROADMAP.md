Along the way, found and fixed a real Mongoose bug in the new `Payment` schema (`razorpayPaymentId`/`method` fields typed as `string | null` without an explicit `type:` override resolved to Mongoose's untyped `Mixed`, crashing NestJS startup with `CannotDetermineTypeError`) — but also **discovered a separate, pre-existing, codebase-wide issue**: every ref field across every schema (`@Prop({ type: Types.ObjectId, ref: '...' })`, using `Types` imported from `'mongoose'`) actually resolves to `Mixed` rather than `ObjectId` under Mongoose 9, because `mongoose.Types.ObjectId` and `mongoose.Schema.Types.ObjectId` are no longer the same reference identity in v9 (confirmed via a minimal repro). The app has worked despite this only because every ID is consistently passed around as a plain string end-to-end, never a real `ObjectId` instance — but it means `.populate()` has never worked anywhere in this app, IDs are stored as raw strings instead of proper BSON ObjectIds, and any future code that constructs a real `ObjectId` for one of these fields will silently get zero query results. Flagged as a separate background task rather than fixed inline here, since it touches every schema file in the project, not just payments |
